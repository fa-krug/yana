# Force Reload for Feeds and Articles Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a "Force reload" gesture action to feeds (re-import every article the source offers, bypassing the intake-window and daily-cap filters) and to articles (a true single-article re-fetch from the article's URL), leaving the existing incremental "Update" actions untouched.

**Architecture:** A `force` flag threads through `AggregationService.aggregate(feed:force:)` and a new `FeedConfig(feed:collectedToday:force:)` initializer; when set it skips the intake-window filter and raises `dailyLimit` to `Int.max` so neither the service cap nor the aggregator's internal `prefix` truncates. A new `Aggregator.refetch(_:)` protocol method (default `nil`, overridden in `FullWebsiteAggregator` to reuse the existing per-article `enrich` path) powers `AggregationService.forceReload(article:)`, which falls back to a forced feed reload when an aggregator returns `nil`. Two SwiftUI swipe buttons call the new service methods.

**Tech Stack:** Swift 6 (strict concurrency, `@MainActor`), SwiftUI, SwiftData, Swift Testing (`import Testing`), SwiftSoup. Built via `xcodebuild`/Xcode (project generated by XcodeGen).

## Global Constraints

- Platform: iOS 26.0+; Swift 6 strict concurrency with `@MainActor` annotations throughout.
- SwiftData is the source of truth; views read via `@Query`, only `AggregationService` writes.
- Article dedup key is `(feed, identifier)`; upsert MUST preserve `createdAt` (timeline position) and Starred state — handled already by `ArticleUpsert.apply`, do not change it.
- Existing "Update" actions (`AggregationService.update(feed:)` / `update(article:)`) MUST remain behaviorally unchanged.
- ALL new user-facing strings MUST be added to `Yana/Resources/Localizable.xcstrings` with a German (`de`) translation marked `"state" : "translated"`. German uses Apple's localization style (infinitive for actions; no "Du"/"Sie").
- Tests use the Swift Testing framework (`@Test`, `#expect`, `#require`) and `@MainActor` where they touch SwiftData.
- Build/test command: `xcodebuild -scheme Yana -destination 'platform=iOS Simulator,name=iPhone 17' test`

```swift
    /// Force reload a single article: re-fetch its content directly from the source when the
    /// aggregator supports it (`refetch`), upserting in place (content refreshed; createdAt +
    /// Starred preserved). Falls back to a forced reload of the parent feed when the aggregator
    /// cannot re-fetch a lone item (e.g. RSS / social sources) or the re-fetch fails.
    @discardableResult
    func forceReload(article: Article) async -> Int {
        guard let feed = article.feed else { return 0 }
        lastRunFailures = []
        isUpdating = true
        defer { isUpdating = false }

        let config = FeedConfig(feed: feed, collectedToday: 0)
        let credentials = AggregatorCredentials.resolved()
        guard let aggregator = makeAggregator(config, credentials) else {
            return await forceReload(feed: feed)
        }
        let seed = AggregatedArticle(
            title: article.title, identifier: article.identifier, url: article.url,
            rawContent: article.rawContent, content: article.content, date: article.date,
            author: article.author, iconURL: article.iconURL
        )
        let refreshed: AggregatedArticle?
        do {
            refreshed = try await aggregator.refetch(seed)
        } catch {
            refreshed = nil
        }
        guard let refreshed else {
            return await forceReload(feed: feed)
        }
        let processed = await currentAIProcessor().process([refreshed], ai: config.options.ai)
        let inserted = ArticleUpsert.apply(processed, to: feed, starredTag: starredTag(),
                                           context: context, now: now())
        cleanupAndSave()
        return inserted
    }
```

## Notes / deviations from the spec

- **Comics get the real single-article re-fetch.** The spec listed the comic scrapers as fallback-only, but `ComicAggregator` subclasses `FullWebsiteAggregator` and its `enrich` is fully URL-driven, so they inherit the real `refetch` for free (no extra code, and a correct per-comic re-fetch). `feedContent`, `podcast`, `reddit`, and `youtube` still fall back as designed.
- **`update(article:)` is kept, not replaced.** The spec mentioned replacing the placeholder `update(article:)`; instead it is left intact so the existing blue "Update" button on articles keeps its current behavior (re-run parent feed), and the new "Force reload" button uses `forceReload(article:)`. This honors the global constraint that existing "Update" actions stay unchanged.
```
